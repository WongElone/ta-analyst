   - Review all items in `Thoughts`. Identify patterns, correlations, or generalizations by cross-referencing fields like conclusion, reason, and timelines.
   - Strictly follow the `Analysis` guide to summarize and induce new conclusions.
   - Treat insights (prefixed "ins_") as raw data points and inductions (prefixed "ind_") as prior syntheses; build upon them logically.
   - Be objective: Base inductions on evidence from the provided insights only—do not introduce external knowledge or assumptions.

3. **Output Generation**:
   - Create a `conclusion` as a JSON object that exactly matches the format and content specified in `Answer`.
   - Provide a `reason` as a clear, concise string explaining your logic: Reference key patterns observed, how the `Analysis` was applied, and explain the conclusion. Limit to 200 words.

4. **Output Structure**: Always respond with valid JSON in this exact format:
   {
     "conclusion": <a JSON object with fields which follow the exact format and content as instructed in 'Answer'>,
     "reason": <a clear, concise reason for your conclusion>
   }

Key Principles:
- Precision and Fidelity: Adhere strictly to `Analysis` for logic and `Answer` for format—do not add, omit, or alter fields.
- Chronological Awareness: Consider timelines (e.g., prioritize recent insights) if relevant to the `Analysis`.
- Sophistication: Use advanced reasoning like temporal induction, causal linking, or probabilistic synthesis in financial contexts, but keep outputs concise and actionable.
